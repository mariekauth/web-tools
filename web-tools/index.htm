Object Oriented Model Writer

Examples

Procedural programming, also known as water-fall, gives the computer instruction line by line. It is faster to write initially, but often does not lend itself well to code re-use.

An Object Oriented approach often allows you to write things once, but use them multiple times. It takes a little longer to write initially, but due to the re-usablity of the code ultimately makes development a bit faster and more reliable.

The first tool I wrote allows me to write the PHP Models using the PSR Standards from PHP-Fig.

Next

I decided that this would be a great project to put on github to demonstrate the use of Version Control Software as well as Team Collaboration. The server side object that currently writes the PHP Objects will be useful for actually writing the model files in a development folder before deploying. However, the working demonstration lends itself better to being parsed completely in JavaScript.

Currently support is only for writing PHP Models and makes a call to the server. We are going to update the code to be parsed entirely in JavaScript with no need to make a call back to the server. My daughter, Shelby Kauth, and I have chosen to work on this project together.

How this came about

On Monday I found myself unemployed and without an up to date portfolio. So, I decided to write a tool that would allow to track the jobs I am interested in, and each interaction with the contacts. This would make a great portfolio project as well as a useful tool that has been requested of me several times in the past.

To demonstrate my abilities to write in Object Oriented PHP using an MVC layout I started by creating the company model.

Having only a text editor as a development tool on this machine I realize that this is going to take a while to create each model along with the proper comments that will allow intellisense from a more robust editor to work correctly.

So, I quickly whipped up a controller that would write the model if sent an array.
