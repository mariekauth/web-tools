Object Oriented Model Writer

Examples

Procedural programming, also known as water-fall, gives the computer instruction line by line. It is faster to write initially, but does not lend itself well to code re-use.

An Object Oriented approach often allows you to write things once, but use them multiple times. It takes a little longer to write initially, but due to the re-usablity of the code ultimately makes development a bit faster and more reliable.

The first tool I wrote allows me to write the PHP Models using the PSR Standards from PHP-Fig for the symfony framework.

Next

I decided that this would be a great project to put on github to demonstrate the use of Version Control Software as well as Team Collaboration. The server side object that currently writes the PHP Objects will be useful for actually writing the model files in a development folder before deploying. However, the working demonstration lends itself better to being parsed completely in JavaScript.

Currently support is only for writing PHP Models and makes a call to the server. We are going to update the code to be parsed entirely in JavaScript with no need to make a call back to the server. My daughter, Shelby Kauth, and I have chosen to work on this project together.
